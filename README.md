# fund-经理

一个运行在 **Trae IDE** 上的多智能体协作基金管理系统。通过 MCP 实时行情 + 三层智能体流水线，对持仓基金进行系统化分析并输出操作建议。

## 能做什么

**说一句「全面分析」，三位 AI 智能体接力完成一次完整的投资复盘：**

| 你想知道的 | 它怎么给你答案 |
|------|------|
| 今天该不该操作？买什么？买多少？ | 索引数 → 板块 → 基金净值 → 估值分位 → 方向评分 → 红绿灯 → 操作清单，**每笔建议附完整逻辑链** |
| 仓位有没有隐患？ | 大类资产偏离度 + 单方向集中度 + 基金质量，**三维诊断，红灯自动标出** |
| 现在买贵不贵？时机对吗？ | 盘中实时指数 / 资金流 / 北向 + T-1 结算净值交叉验证，**不凭感觉** |
| 为什么推荐这个方向而不是别的？ | 持仓方向 + 非持仓方向 全覆盖评分，**逻辑、估值、资金三维打分，不遗漏机会** |
| 会不会买到追高？会不会超仓？ | 每笔建议过买入时机四问 → 三道仓位校验 → 三层递进红绿灯，**三重过滤后才放行** |

> 简单说：替你做完数据收集、分析、纠偏，输出一份**可直接执行的操作清单**，你只需要决定"做不做"。

```
├── 智能体_数据采集员.md        # 智能体①：MCP数据采集、净值更新、新闻搜索
├── 智能体_分析决策员.md        # 智能体②：执行完整分析框架
├── 智能体_报告整合员.md        # 智能体③：格式编排、完整性校验、保存指引
├── mcp/
│   └── .mcp.json              # MCP 服务配置（仅需 stock-sdk-mcp）
├── 持仓情况/
│   ├── 持仓情况.template.md     # 持仓模板（拉取后复制为 持仓情况.md 填写）
│   └── 待确认操作.template.md   # 待确认操作模板
├── 全面分析/
│   └── 全面分析_YYYY-MM-DD.md  # 历史分析报告存档
├── .trae/
│   └── instructions.md         # Trae IDE 项目级指令（入口引导）
├── .gitignore                  # Git 忽略规则
└── .mcp.json                  # 项目级 MCP 配置（如 IDE 需放在根目录）

> **重要：** 本项目不是可执行程序，是一套**给 AI 读的操作手册**。执行依赖 AI 读取并遵循文件中的指令。详见下方「运行原理」。
```

## 工作流

```
用户说「全面分析」
    │
    └→ fund-经理_SOP.md（调度中心，意图识别 + 流程编排）
         │
         ├→ 持仓数据（持仓情况/持仓情况.md，唯一真相源）
         │
         ├→ ① 数据采集员     MCP + WebSearch
         │
         ├→ ② 分析决策员     读取 fund经理配置_完整版.md（八条逻辑 + 评分体系 + 校验规则）
         │   ├── 资讯全景扫描（三级冲击分级）
         │   ├── 资产配置仪表盘（中枢偏离检查）
         │   ├── 方向扫描（逻辑+估值+资金三维评分）
         │   ├── 综合操作建议（红灯清单+方案+冲突项）
         │   ├── 持仓诊断 & 调入调出
         │   ├── 操作红绿灯（三层递进：该不该动 → 时机对吗 → 怎么动）
         │   └── 执行清单 + 历史验证
         │
         └→ ③ 报告整合员     格式编排 + 31项完整性校验 + 文件保存指引
```

> **SOP** 负责识别用户意图、编排流程、管理持仓数据；**经理配置** 是 ② 的规则引擎，定义了八条逻辑、评分标准、校验规则、输出格式等全部执行细节。两者是上下层关系：SOP 决定"谁来做"，配置决定"怎么做"。

## 运行原理（重要）

本项目**不是可执行程序**，没有 main 函数、没有调度器、没有编译器。

### 实际运作方式

```
用户说"全面分析"
    │
    └→ AI 读取 .trae/instructions.md（项目指令，提示 AI 去找 SOP）
         │
         └→ AI 读取 fund-经理_SOP.md，按其中定义的流程：
              ├── 阶段一：读取 智能体_数据采集员.md → 照做
              ├── 阶段二：读取 fund经理配置_完整版.md → 照做
              └── 阶段三：读取 智能体_报告整合员.md → 照做
```

整个"程序"就是 AI 按顺序阅读 Markdown 文件、执行其中的文本指令。没有代码强制执行任何一个步骤。

### 可靠性的边界

| 保证 | 不保证 |
|------|------|
| ✅ 文件内容完整、规则自洽 | ❌ AI 一定会读到这些文件 |
| ✅ MCP 配置正确、数据通路畅通 | ❌ AI 一定严格遵循所有规则 |
| ✅ 分析框架逻辑严密 | ❌ 不同 AI 模型输出完全一致 |

**本质是一套精心设计的提示词工程**，可靠性取决于 AI 的理解和执行能力，不是代码逻辑的确定性。

### .trae/instructions.md 的作用

这是 Trae IDE 的项目级指令文件。IDE 在启动对话时会自动读取，告诉 AI "遇到'全面分析'时先看 SOP"。它是入口引导，降低了 AI 忽略项目规则的概率，但不是绝对保证。

## 初次使用

### 1. 前置条件

- 安装 [Trae IDE](https://trae.ai)（智能体运行平台）
- 启用 MCP 功能

### 2. 配置 MCP 数据源

项目已预置 `mcp/.mcp.json`，仅需确保 `stock-sdk-mcp` 可用：

```json
{
  "mcpServers": {
    "stock-sdk": {
      "command": "npx",
      "args": ["-y", "stock-sdk-mcp"]
    }
  }
}
```

这个包覆盖了全部盘中数据需求（指数行情、板块涨跌、资金流向、基金净值、北向资金），无需额外配置。

### 3. 填入你的持仓

编辑 `持仓情况/持仓情况.md`，按格式填入你的基金持仓：

```markdown
| 基金代码 | 基金名称 | 持有份额 | 成本净值 | 最新净值 | 最新市值 | 占比 | 浮动盈亏 | 盈亏率 |
|:--------:|----------|:----------:|:------:|:------:|:------:|:------:|:------:|:--:|
| 007466 | 华泰柏瑞红利低波ETF联接A | X | X | X | X | X% | X | X% |
```

余额宝单独标注金额。数据格式详见文件中注释。

### 4. 说一句「全面分析」

在 Trae IDE 对话框中输入「**全面分析**」四个字，基金-经理自动启动三阶段流水线，输出完整报告。

### 5. 常用指令

| 你说 | 触发 |
|------|------|
| `全面分析` | 完整三阶段流程（数据→分析→报告） |
| `快速分析` | 跳过数据采集，沿用已有数据直接分析 |
| `更新持仓` | 用你提供的数据覆盖持仓文件 |
| `加仓 XX 基金 X 元` | 记录操作意向，T+1 净值公布后确认 |
| `已买入 XX 元` | 扣余额宝 + 标待确认，等净值后更新 |

## 分析框架核心规则

| 规则 | 说明 |
|------|------|
| 逻辑强度 = 长线逻辑 | 与短线走势无关，"这几天没涨" ≠ 逻辑弱 |
| 买入时机四问 | 每笔买入必须回答：方向逻辑 / 为什么现在 / 跌了怎么办 / 不买会错过什么 |
| T日买入禁则 | 只做两件事——扣余额宝 + 标"待净值确认"。不碰份额/市值/占比 |
| 三层递进 | 第三层说可买 + 第四层说绿灯 → 第五层才执行 |
| 仓位校验 | 每笔建议过三关：单方向上限 / 大类资产红线 / 现金充足 |
